## 一、DRY 违反 — 重复定义

### 1.1 CSS 变量在基础声明和 `.light` 块中完全重复

**文件：** `app/app.vue:90-152`

基类 `#container`（第 90 行）已将所有亮色主题值作为默认值，而 `#container.light`（第 127 行）又逐字重复了完全相同的 20 行变量定义。零差异，纯冗余。

```less
// 第 90 行 — 基类默认值
#container {
  --bg-primary: #f0f2f5;
  --bg-secondary: #ffffff;
  // ...（共 20 个变量）
}

// 第 127 行 — 与上面完全相同的值
#container.light {
  --bg-primary: #f0f2f5;
  --bg-secondary: #ffffff;
  // ...（同上，一字不差）
}
```

**修复建议：** 删除 `#container.light` 块中与基类重复的所有变量，`.light` 块只保留 `--bg-image`（亮色特有）。

### 1.2 `#container` 声明块被拆成两段

**文件：** `app/app.vue:90-124` 和 `:193-207`

同一个选择器 `#container` 的属性分散在两处，中间插入了 `.light`、`.dark`、`@media` 块。阅读时需要上下跳转才能看清完整样式。

**修复建议：** 合并为一个 `#container` 声明块。

## 二、暗色主题适配遗漏

### 2.1 分类区块图标无暗色滤镜

**文件：** `app/components/CategorySection.vue:13`

```html
<img :src="categoryItem.iconSvg" :alt="categoryItem.category">
```

`<img>` 无 `filter: var(--icon-filter)`，与之前侧边栏的 bug 同源。暗色主题下分类标题左侧的深色图标不可见。

**修复建议：** 添加 `filter: var(--icon-filter)` 到 `.section-icon img`。

### 2.2 搜索引擎图标无暗色滤镜

**文件：** `app/components/SearchBar.vue:35、:66`

搜索引擎选择按钮和下拉面板中的图标均无暗色滤镜。

```html
<!-- 搜索栏引擎图标 -->
<img :src="currentSearchEngine.logo" ...>

<!-- 下拉面板引擎图标 -->
<img :src="item.logo" ...>
```

**修复建议：** 为 `.search-engine-btn img` 和 `.engine-item img` 添加 `filter: var(--icon-filter)`。

### 2.3 网站卡片 logo 图标无暗色滤镜

**文件：** `app/components/WebsiteCard.vue:14`

```html
<img v-else :src="websiteItem.logo" ... class="card-logo">
```

网站 logo 图片没有暗色滤镜，部分深色 logo 在暗色背景下不可见。

**修复建议：** 为 `.card-logo` 添加 `filter: var(--icon-filter)`。

## 三、逻辑问题

### 3.1 `useSearch` 中 watch 计算了建议但未使用

**文件：** `app/composables/useSearch.js:63-73`

```javascript
watch(searchContent, (newVal, oldVal) => {
  if (newVal.trim() === '' || newVal === oldVal) return
  const cacheList = readSearchHistory()
  const suggestions = []
  cacheList.forEach((item) => {
    if (item.indexOf(newVal) !== -1) {
      suggestions.push(item)
    }
  })
  // suggestions 从未被使用 — 纯无效计算
})
```

每次输入都读取 localStorage、遍历匹配、构建数组，但结果没有赋给任何响应式变量，也不渲染到 UI。这是残留的死代码。

**修复建议：** 删除整个 watch，或将其结果暴露给模板实现搜索建议 UI。

### 3.2 `useSearch` 中用 `setTimeout(0)` 代替 `nextTick`

**文件：** `app/composables/useSearch.js:80-82`

```javascript
setTimeout(() => {
  searchInputDom.value?.focus()
}, 0)
```

`setTimeout(0)` 是不精确的 hack，正确做法是用 Vue 的 `nextTick` 等待 DOM 更新完成。

**修复建议：** 改为 `nextTick(() => { searchInputDom.value?.focus() })`。

### 3.3 `useTheme` — localStorage 被重复写入

**文件：** `app/composables/useTheme.js`

`watch` 设置了 `{ immediate: true }`，组件初始化时立即触发一次，把 `'light'` 写入 localStorage。然后 `onMounted` 又读取 localStorage 并再次设置。首次访问时，watch 的 immediate 写入了无意义的默认值（而 `onMounted` 中可能读到 old saved value 又覆写）。

**修复建议：** 去掉 `immediate: true`，将初始同步逻辑全部放在 `onMounted` 中：

```javascript
onMounted(() => {
  const saved = window.localStorage.getItem('nowThemeType')
  if (saved === 'dark' || saved === 'light') {
    currentTheme.value = saved
  }
  // 同步 class（watch 在 onMounted 之后触发，此时 DOM 已就绪）
})
watch(currentTheme, (theme) => { /* 只做 class 切换 + 持久化 */ })
```

### 3.4 `SideNav` 中 `:target` 伪类无效

**文件：** `app/components/SideNav.vue:100`

```css
&:target,
&:active {
  border-left-color: var(--accent);
  ...
}
```

`.nav-link` 是 `<a>` 元素，没有 `id` 属性，CSS `:target` 永远不会匹配它。`:target` 匹配的是 `id` 等于 URL hash 的元素。这条规则从未生效过。

**修复建议：** 删除 `&:target`，如需高亮当前锚点对应的导航项，需用 JS 监听 hash 变化。

## 四、代码质量问题

### 4.1 HTML 语义错误 — `<header>` 嵌套 `<header>`

**文件：** `app/app.vue:31-32`

```html
<header id="container-header">
  <div id="header-logo">
    <header id="logo-box">   ← <header> 嵌套在 <header> 内
```

HTML 规范禁止 `<header>` 嵌套另一个 `<header>`。内层应改为 `<div>`。

### 4.2 `resolveAssetPaths` 原地修改导入的常量

**文件：** `app/data/WebSiteData.js:3-11`

```javascript
function resolveAssetPaths(data) {
  if (Array.isArray(data)) {
    data.forEach(resolveAssetPaths)
  } else if (data && typeof data === 'object') {
    if (data.iconSvg) data.iconSvg = getAssetUrl(data.iconSvg)  // 直接修改原对象
    if (data.logo) data.logo = getAssetUrl(data.logo)
    ...
  }
}
```

对导入的静态数据常量进行原地修改。虽然功能正常，但违反了"数据不可变"原则，如果多处导入可能有意外副作用。

**修复建议：** 使用 `structuredClone()` 或展开运算符先复制再修改。

### 4.3 全局 CSS `*` 选择器 `color: inherit` 过于激进

**文件：** `app/assets/css/global.css:46-52`

```css
* {
    color: inherit;
}
```

给每个元素设置 `color: inherit` 意味着 `<input>`、`<button>`、`<a>` 等原生元素的行为被覆盖。浏览器默认为这些元素提供平台原生颜色，此重置可能导致非预期表现。

**修复建议：** 改为只对需要的元素设置，或使用更保守的 reset。
